# Scan Modes

## Quick Scan

Fast preliminary pre-open review. It is designed to help decide whether a project should be opened quickly.

Quick Scan checks high-risk entry points first and may show more warnings because it uses less context.

## Deep Scan

Slower and broader behavior/context review. It is better before trusting, running, or publishing a project.

Deep Scan reads more context and may reduce false positives. In v0.6.0 the console progress includes elapsed time and ETA-style estimates so users can decide whether to continue waiting.
